# 화평법 화학물질 규제DB 업데이트 명령서

> **트리거:** 사용자가 "화학물질 규제DB 업데이트", "K-REACH DB 갱신", "화평법 목록 다시 받아줘" 등의 요청을 하면 이 문서의 절차를 순서대로 실행하라.

---

## 1단계 — 신규 엑셀 파일 다운로드

아래 URL에서 전체 화학물질 목록을 엑셀로 다운로드하라.

```
https://kreach.mcee.go.kr/repwrt/mttr/kr/mttrList.do?searchCasnoKeyword=&searchIdentiNoKeyword=&searchEnKeyword=&searchKrKeyword=
```

**실행 방법 (Chrome MCP 사용 시):**
1. 위 URL로 이동
2. 검색어 입력 없이 **[검색]** 버튼 클릭 → 전체 물질 조회 확인
3. **[엑셀다운로드]** 버튼 클릭
4. 다운로드 완료 후 파일 경로 확인: `C:\Users\...\Downloads\화학물질_YYYYMMDD.xlsx`

> 파일명의 날짜(`YYYYMMDD`)는 다운로드 실행 당일 자동 생성된다.

---

## 2단계 — 다운로드 파일 구조 확인 및 검증

다운로드한 파일을 열어 아래 기준으로 구조를 검증하라.

### 예상 구조

- **시트명:** `화학물질_YYYYMMDD` (날짜가 파일명과 동일)
- **헤더 행:** 1행
- **데이터 시작:** 2행
- **총 컬럼:** 20개 (A~T열)
- **예상 데이터 행 수:** 47,000건 이상 (이전 다운로드 대비 증감 확인)

### 헤더 검증 — 1행의 컬럼명이 아래와 정확히 일치해야 한다

| 열 | 기대 컬럼명 |
|----|------------|
| A | NO |
| B | CAS번호 |
| C | 영문명 |
| D | 국문명 |
| E | 기존 |
| F | 급성·만성·생태 |
| G | 사고대비 |
| H | 제한/금지/허가 |
| I | 중점 |
| J | 잔류 |
| K | 유해특성분류 및 혼합물 함량기준(%) |
| L | 등록대상기존화학물질 |
| M | 기존물질여부 |
| N | 인체급성유해성 |
| O | 인체만성유해성 |
| P | 생태유해성 |
| Q | 제한물질 |
| R | 금지물질 |
| S | 허가물질 |
| T | 사고대비물질 |

> **컬럼 수 또는 헤더명이 다르면 작업을 중단하고 사용자에게 알려라.**  
> 환경부가 파일 구조를 변경했을 가능성이 있다.

## 3단계 — 컬럼별 데이터 변환 규칙

각 열의 데이터를 아래 규칙에 따라 읽고 처리하라.

### 식별 컬럼 (A~E열) — 그대로 유지

| 열 | 컬럼명 | 처리 방법 |
|----|--------|----------|
| A | NO | 정수로 변환 |
| B | CAS번호 | 문자 그대로 유지 (하이픈 포함) |
| C | 영문명 | 문자 그대로 유지. 이명은 `;`으로 구분됨 |
| D | 국문명 | 문자 그대로 유지. 공란 허용 |
| E | 기존 | `KE-XXXXX` 형식 검증. 공란 허용 |

### 규제 분류 컬럼 (F~J열) — 규칙 기반 해석

**F열 (급성·만성·생태)** — 형식: `{유해성종류}{고시연도}-{고시번호}({차수})`

유해성 종류 접두어로 아래 분류를 파악한다:

| 접두어 패턴 | 분류 |
|------------|------|
| `인체급성`만 포함 | 인체급성유해성 물질 |
| `인체만성`만 포함 | 인체만성유해성 물질 |
| `생태`만 포함 | 생태유해성 물질 |
| `인체급성·인체만성·생태` | 3종 복합 유해성 |
| `인체만성·생태` | 인체만성 + 생태 복합 |
| `인체급성·생태` | 인체급성 + 생태 복합 |

**G열 (사고대비)** — 숫자이면 사고대비물질 번호. 공란이면 미해당.

**H열 (제한/금지/허가)** — 형식: `{연도}-{구분코드}-{번호}`
- 구분코드 `5` = 제한물질
- 구분코드 `4` = 금지물질
- 구분코드 `3` = 허가물질
- 쉼표로 복수 고시 번호가 연결될 수 있음

**I열 (중점)** — `별표-숫자` 형식. 공란이면 미해당.

**J열 (잔류)** — 공란이면 미해당.

### 서술형 텍스트 컬럼 (K열) — 파싱 후 N~S열 값 검증

K열은 유독물질 혼합물 함량 기준을 서술형 텍스트로 담고 있다.  
여러 규제 유형은 ` / `로 구분된다.

**파싱 패턴:**
```
유독물질 : 인체급성유해성 : {N}%        → N열 값
유독물질 : 인체만성유해성 : {O}%        → O열 값
유독물질 : 생태유해성 : {P}%            → P열 값
제한물질 : ... 이를 {Q}% 이상 함유한    → Q열 값
금지물질 : ... 이를 {R}% 이상 함유한    → R열 값
허가물질 : ... 이를 {S}% 이상 함유한    → S열 값
```

> N~S열은 K열에서 파싱된 숫자값이다. K열이 공란이면 N~S열도 모두 공란이어야 한다.  
> 불일치가 발견되면 해당 행을 오류 목록에 기록하라.

### 수치 컬럼 (N~T열) — 숫자 타입 검증

| 열 | 컬럼명 | 허용값 |
|----|--------|--------|
| N | 인체급성유해성 | 양수 숫자 또는 공란 |
| O | 인체만성유해성 | 양수 숫자 또는 공란 |
| P | 생태유해성 | 양수 숫자 또는 공란 |
| Q | 제한물질 | 양수 숫자 또는 공란 |
| R | 금지물질 | 양수 숫자 또는 공란 |
| S | 허가물질 | 양수 숫자 또는 공란 |
| T | 사고대비물질 | 양수 정수 또는 공란. G열 값과 동일해야 함 |

### 기존물질여부 (M열) — 허용값 검증

- `Y` = 기존화학물질
- `N` = 신규화학물질
- 공란 = 미분류 (허용)
- 그 외 값이 있으면 오류로 기록

## 4단계 — 변환 출력 형식

처리 완료 후 아래 형식으로 요약 보고하라.

```
[화학물질 규제DB 업데이트 완료]

파일명    : 화학물질_YYYYMMDD.xlsx
총 건수   : XX,XXX건
기존물질  : XX,XXX건 (M열=Y)
신규물질  : X,XXX건 (M열=N)

규제물질 현황:
  - 인체등유해성 (F열): X,XXX건
  - 사고대비물질 (G열): XXX건
  - 제한/금지/허가 (H열): XXX건
  - 중점관리물질 (I열): XXX건
  - 잔류성오염물질 (J열): XX건

오류/이상 행: X건 (있을 경우 목록 첨부)
이전 대비 증감: +XXX건 / -XXX건
```

## 5단계 — 이상 상황 처리 기준

| 상황 | 조치 |
|------|------|
| 컬럼 수가 20개가 아님 | **즉시 중단**, 사용자에게 구조 변경 가능성 알림 |
| 헤더명 불일치 | **즉시 중단**, 변경된 컬럼명 목록을 사용자에게 보고 |
| 총 건수가 40,000건 미만 | 경고 표시 후 계속 진행 |
| T열 값이 G열과 다른 행 발견 | 해당 행을 오류 목록에 기록 |
| K열 있으나 N~S열 모두 공란인 행 | 파싱 실패로 기록, 수동 확인 요청 |
| 사이트 접속 불가 | 사용자에게 안내 후 수동 다운로드 요청 |

## 결과물 저장 경로

업데이트 실행 후 모든 결과물은 아래 경로에 날짜별 폴더를 만들어 저장하라.

**저장 기본 경로:**
```
outputs\화평법_규제DB_결과물\YYYYMMDD\
```

**저장 파일 목록:**

| 파일명 | 내용 |
|--------|------|
| `화학물질_YYYYMMDD.xlsx` | KREACH에서 다운로드한 원본 엑셀 파일 |
| `업데이트_보고서_YYYYMMDD.md` | 검증 결과 및 규제물질 현황 통계 보고서 |

**폴더 구조 예시:**
```
