# K-REACH 화학물질 규제DB 업데이트 스킬

이 스킬은 세 가지 작업을 순서대로 처리한다.
1. KREACH 사이트에서 최신 엑셀 파일 다운로드
2. 다운로드된 파일의 구조 검증
3. K열 파싱 → N~T열 추가 변환 후 `_변환` 파일로 저장

세부 절차는 아래 참조 파일에 있다.
- 다운로드 방법 → `references/다운로드_방법.md`
- 엑셀 구조 및 변환 규칙 → `references/엑셀_구조_가이드.md`
- 변환 스크립트 → `scripts/transform.py`

---

## 실행 순서

요청을 받으면 아래 순서대로 진행하라.

### STEP 1 — 다운로드

`references/다운로드_방법.md`를 읽고 절차를 실행하라.

핵심:
- URL: `https://kreach.mcee.go.kr/repwrt/mttr/kr/mttrList.do?searchCasnoKeyword=&searchIdentiNoKeyword=&searchEnKeyword=&searchKrKeyword=`
- Chrome MCP로 페이지 이동 → [검색] 버튼 클릭 → [엑셀다운로드] 클릭
- 다운로드 파일: `C:\Users\...\Downloads\화학물질_YYYYMMDD.xlsx`

### STEP 2 — 구조 검증

다운로드된 파일(원본, 13열)을 열고 아래를 확인하라.

- 헤더 A~M열 13개 컬럼명이 가이드와 일치하는지 확인
- 불일치 시 **즉시 중단**하고 사용자에게 보고

> 원본 다운로드 파일은 13열(A~M)이다. N~T열은 STEP 3에서 추가된다.

### STEP 3 — 변환 실행 및 결과물 저장

검증 통과 후 아래 순서로 실행하라.

**① 변환 스크립트 실행**

`scripts/transform.py`를 사용해 13열 원본을 20열로 변환한다.

```bash
python3 scripts/transform.py \
  화학물질_YYYYMMDD.xlsx \
  출력경로/화학물질_YYYYMMDD_변환.xlsx
```

변환 내용 (K열 텍스트 → 수치 파싱):

| 추가 열 | 컬럼명 | 파싱 패턴 |
|--------|--------|----------|
| N | 인체급성유해성 | `유독물질 : 인체급성유해성 : X%` |
| O | 인체만성유해성 | `유독물질 : 인체만성유해성 : X%` |
| P | 생태유해성 | `유독물질 : 생태유해성 : X%` |
| Q | 제한물질 | `제한물질 : ... 이를 X% 이상` |
| R | 금지물질 | `금지물질 : ... 이를 X% 이상` |
| S | 허가물질 | `허가물질 : ... 이를 X% 이상` |
| T | 사고대비물질 | G열 값 복사 |

**② 날짜별 폴더에 저장**

```
outputs\화평법_규제DB_결과물\YYYYMMDD\
```

저장 파일:
- `화학물질_YYYYMMDD.xlsx` — 원본 다운로드 파일 (13열)
- `화학물질_YYYYMMDD_변환.xlsx` — 변환 완료 파일 (20열) ← **핵심 결과물**
- `업데이트_보고서_YYYYMMDD.md` — 통계 보고서

**③ 보고서 형식 (`업데이트_보고서_YYYYMMDD.md`)**

```markdown
# 화학물질 규제DB 업데이트 보고서

업데이트 일시: YYYY-MM-DD

## 결과 요약

| 항목 | 값 |
|------|-----|
| 원본 파일 | 화학물질_YYYYMMDD.xlsx (13열) |
| 변환 파일 | 화학물질_YYYYMMDD_변환.xlsx (20열) |
| 총 건수 | XX,XXX건 |
| 기존물질 (M=Y) | XX,XXX건 |
| 신규물질 (M=N) | X,XXX건 |

## 규제물질 현황

| 규제 유형 | 건수 |
|----------|------|
| 인체등유해성 (F열) | X,XXX건 |
| 사고대비물질 (G열) | XXX건 |
| 제한/금지/허가 (H열) | XXX건 |
| 중점관리물질 (I열) | XXX건 |
| 잔류성오염물질 (J열) | XX건 |

## 변환 결과

- N열(인체급성유해성) 파싱: X,XXX건
- T열(사고대비물질) 복사: XXX건
- 파싱 실패 의심 행: X건

## 검증

- 헤더 검증: ✅ 통과 / ❌ 실패
- 이전 대비 증감: +XXX건 / -XXX건
```

보고서 저장 후 채팅창에도 동일 내용을 출력하라.

### 이상 상황

| 상황 | 조치 |
|------|------|
| 컬럼 수가 20개 아님 | 즉시 중단, 사용자 보고 |
| 헤더명 불일치 | 즉시 중단, 변경 목록 보고 |
| 총 건수 40,000건 미만 | 경고 후 계속 |
| 사이트 접속 불가 | 수동 다운로드 요청 |
